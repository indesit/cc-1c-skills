# Runbook — розгортання і експлуатація скілів на PRODUCTION-сервері

Тулсет розробляється і обкатується на тестовому сервері (копія бази). На прод їде **тільки тегнутий стан** репозиторію, перевірений там. Цей runbook — порядок розгортання і правила роботи на проді.

## 1. Розгортання (одноразово)

1. Скопіювати/клонувати репозиторій скілів і запустити від адміністратора:
   ```powershell
   powershell -NoProfile -ExecutionPolicy Bypass -File bootstrap\bootstrap.ps1 -WorkspaceDir <робоча папка>
   ```
   Скрипт ідемпотентний: перевіряє передумови (git, python, платформа, rac, sqlcmd), клонує/оновлює репозиторій, створює junction `.claude\skills`, шаблон `.v8-project.json`, підказує установку RAS.
2. Заповнити `.v8-project.json` (**тільки `passwordEnv`, ніякого plaintext**), поставити Machine-змінну з паролем.
3. Установити службу RAS (інструкцію друкує bootstrap; див. також `srv-info` SKILL.md).
4. Верифікація read-only: `/db-list` → `/srv-info` → `/log-analyze` → `/cf-check`. Жодних write-операцій у перший день.

## 2. Оновлення скілів на проді

```powershell
git -C <repos>\cc-1c-skills fetch
git -C <repos>\cc-1c-skills checkout <тег, перевірений на тестовому сервері>
```

Ніколи не редагувати скіли прямо на проді — зміни тільки через тестовий сервер → коміт → тег → checkout.

## 3. Правила write-операцій на проді

| Операція | Обов'язково перед нею |
|----------|----------------------|
| `db-update`, `db-load-*`, завантаження розширення | `/db-backup` (sql, COPY_ONLY) → перевірити маніфест; `/srv-sessions` list → погодити з користувачами → terminate |
| Оновлення конфігурації вендора | `/cfe-compat` КОЖНОГО розширення проти дампа нового релізу; ERRORS лагодити до оновлення |
| Будь-яка зміна | `/cf-check` після завантаження, до `db-update`; smoke-тест РМК після (runbook у робочій папці) |
| `db-restore` | Тільки з явного письмового підтвердження власника; спочатку restore в копію |

Регулярно (щотижня): `/cf-drift` еталон vs свіжий дамп — дрейф або приймається в репозиторій, або розслідується.

## 4. Заборонено на проді

- Plaintext-паролі в будь-яких файлах.
- `/RestoreIB` / `RESTORE DATABASE` поверх робочої бази без свіжого бекапа поточного стану.
- Команди `/LoadExtension`, `/DumpExtension` (висять headless на 8.3.19).
- Експерименти "подивитись що буде" — для цього є тестовий сервер.
