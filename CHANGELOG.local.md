## local-v2026.06.13

Інтеграція з Hermes (MCP), збагачення зовнішніми скілами, виправлення.

### Hermes MCP bridge
- `mcp_bridge/` — MCP-сервер, що віддає Hermes 6 структурованих tools поверх перевірених скриптів: `project_info`, `cf_drift`, `db_dump_xml`, `db_backup`, `srv_info`, `srv_sessions`. Безпека: редакція секретів, `execute=false` (preview) за замовчуванням для важких/мутуючих, terminate сеансів — лише з `confirmation_token='TERMINATE_SESSIONS'`.
- `mcp_bridge/requirements.txt` — `mcp==1.26.0` (збігається з venv Hermes).
- Перевірено: e2e проти живого сервера (`project_info`, `srv_info`), 5/5 юніт-тестів, без витоку пароля.

### Нові/імпортовані скіли (з Desko77/claude-code-skills-1c, MIT)
- `rules/` — 25 знаннєвих BSL-правил (стандарти, антипатерни, оптимізація запитів, патерни розширень/форм, тестування). `rules/README.md` позначає 7 правил, що покладаються на відсутні в нас MCP-сервери. Атрибуція: `rules/LICENSE.Desko77.md`.
- `v8unpack-cf` — розпаковка/збірка CF/CFE/EPF без платформи (`v8unpack` 1.2.6). Додано вимогу `PYTHONUTF8=1` (інакше падає на кирилиці на Windows). Перевірено на `КассирАвтоКасса.cfe` (168 файлів).

### Виправлення

## local-v2026.06.12

Перший консолідований реліз інфраструктурних скілів (Фази 1–3) + промоушн-комплект.

### Нові скіли

- **cf-check** — платформена перевірка `/CheckConfig` + `/CheckModules` (read-only)
- **db-backup** — бекап ІБ: `.dt` через Конфігуратор або online MS SQL `COPY_ONLY` + JSON-маніфест
- **db-restore** — відновлення з `.dt`/`.bak`; відмова без `-IAmSure` (exit 2), показ маніфесту
- **srv-info** — огляд кластера через `rac` (кластер, ІБ, процеси, сеанси)
- **srv-sessions** — список/завершення сеансів; terminate під `-IAmSure`; сценарій "звільнити базу"
- **cfe-compat** — статична перевірка сумісності розширення з дампом конфігурації (запозичені об'єкти/форми, перехоплені методи, сигнатури, виклики базових функцій)
- **log-analyze** — аналіз журналу реєстрації (старий формат `.lgf`/`.lgp`): помилки за період, топи, деталі
- **cf-drift** — контроль дрейфу: еталонні XML-сорси vs свіжий дамп живої бази (sha256)

### Зміни існуючих скілів

- `passwordEnv` у всіх скриптах із підключенням до БД (10 скілів, py+ps1): пароль з env-змінної (process → HKCU → HKLM), plaintext більше не потрібен; `-Password` зберігає пріоритет
- Маскування пароля в echo команд Designer (`/P"********"`)
- Поле `passwordEnv` задокументовано в `db-list` SKILL.md
- fix(skd-edit): кирилічні unicode-escape винесені з виразів f-string — скрипт не компілювався на Python 3.11

### Інфраструктура

- `bootstrap/bootstrap.ps1` — ідемпотентне розгортання тулсета на новому сервері
- `docs/PROD-RUNBOOK.md` — порядок розгортання і правила write-операцій на проді

### Перевірено на тестовому сервері (SD-218-215-XRB, BAF 8.3.19.1529, MS SQL)

- e2e: дамп розширення з `-PasswordEnv` (пароль із Machine-реєстру, у логах замаскований)
- повний цикл db-backup → мутація даних → db-restore → відкат підтверджено (одноразова SQL-база)
- `cf-check` модулів живого розширення; `srv-sessions` завершив реальний клієнтський сеанс
- `cfe-compat`: 2 розширення COMPATIBLE без фальшивих спрацювань; саботаж → INCOMPATIBLE
- `log-analyze`: 2728 записів історичного журналу розпарсено
- `cf-drift`: знайдено реальний дрейф (13 файлів) між еталоном і живою базою
