# /cf-drift — Контроль дрейфа конфигурации

Сравнивает два дерева XML-выгрузки конфигурации по sha256 и отвечает на вопрос: **«совпадает ли живая база с эталонными исходниками?»** `ConfigDumpInfo.xml` игнорируется по умолчанию (метаданные выгрузки, различаются всегда).

## Типовой сценарий

1. Свежий дамп живой базы: `/db-dump-xml` в временный каталог (`-Mode Full`).
2. `cf-drift -Reference <configSrc из .v8-project.json> -Actual <временный каталог>`.
3. `NO DRIFT` — база совпадает с репозиторием. `DRIFT DETECTED` — покажи списки пользователю; изменения нужно либо принять в репозиторий (перенести в configSrc и закоммитить), либо расследовать.

Подходит для регулярного запуска (например, еженедельно через планировщик/`/schedule`).

## Команда

Движок — Python (`scripts/cf-drift.py`); ps1 — обёртка.

```powershell
python .claude/skills/cf-drift/scripts/cf-drift.py -Reference <dir> -Actual <dir> [-Json report.json]
```

### Параметры

| Параметр | Обязательный | Описание |
|----------|:------------:|----------|
| `-Reference <путь>` | да | Эталонное дерево (configSrc) |
| `-Actual <путь>` | да | Свежий дамп живой базы |
| `-Json <файл>` | нет | Сохранить отчёт в JSON |
| `-MaxList <N>` | нет | Сколько путей печатать на категорию (50) |
| `-NoDefaultIgnore` | нет | Не игнорировать ConfigDumpInfo.xml |

## Коды возврата

| Код | Описание |
|-----|----------|
| 0 | NO DRIFT |
| 1 | DRIFT DETECTED |
| 2 | Ошибка (каталог не найден) |

## Примеры

```powershell
python .claude/skills/cf-drift/scripts/cf-drift.py -Reference "src\baf-config-dump" -Actual "C:\tmp\fresh-dump" -Json drift.json
```
