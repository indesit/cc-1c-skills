# /db-restore — Восстановление базы из бэкапа

**ДЕСТРУКТИВНАЯ операция: полностью перезаписывает целевую базу.**

## Обязательный протокол безопасности

1. **ВСЕГДА** спроси подтверждение через AskUserQuestion, показав: целевую базу, файл бэкапа, дату бэкапа из манифеста (`<файл>.manifest.json`, если есть).
2. Скрипт откажется работать без флага `-IAmSure` — добавляй его **только после** явного подтверждения пользователя.
3. Если цель — рабочая (production) база: предложи сначала восстановить в копию (отдельную тестовую базу) и проверить.
4. Перед восстановлением поверх живой базы предложи сделать свежий бэкап текущего состояния (`/db-backup`).

## Режимы

| Режим | Механизм | Примечания |
|-------|----------|------------|
| `dt` | Конфигуратор `/RestoreIB` | Требует монопольного доступа; сеансы должны быть завершены |
| `sql` | `RESTORE DATABASE ... WITH REPLACE` | Переводит базу в SINGLE_USER (принудительно разрывает соединения), восстанавливает, возвращает MULTI_USER |

## Команда

```powershell
powershell.exe -NoProfile -File .claude/skills/db-restore/scripts/db-restore.ps1 <параметры>
```

### Параметры скрипта

| Параметр | Обязательный | Описание |
|----------|:------------:|----------|
| `-Mode <dt\|sql>` | да | Режим восстановления |
| `-InputFile <путь>` | да | Файл бэкапа (`.dt` или `.bak`) |
| `-IAmSure` | да | Подтверждение деструктивной операции (только после согласия пользователя!) |
| `-V8Path <путь>` | dt | Каталог bin платформы |
| `-InfoBasePath <путь>` | dt* | Файловая база |
| `-InfoBaseServer <сервер>` | dt* | Сервер 1С |
| `-InfoBaseRef <имя>` | dt* | Имя базы на сервере 1С |
| `-UserName <имя>` | нет | Пользователь 1С (dt) |
| `-Password <пароль>` | нет | Пароль (dt) |
| `-PasswordEnv <имя>` | нет | Переменная окружения с паролем — вместо `-Password` |
| `-SqlServer <адрес>` | нет | Сервер MS SQL (по умолчанию `localhost`) |
| `-SqlDatabase <имя>` | sql | Целевая база MS SQL (будет ПЕРЕЗАПИСАНА) |

## После выполнения

Сообщи статус. После восстановления рекомендуй: открыть базу, проверить ключевые данные, при серверном restore — проверить, что пользователи могут подключиться.

## Примеры

```powershell
# Восстановление MS SQL базы (после подтверждения пользователя!)
powershell.exe -NoProfile -File .claude/skills/db-restore/scripts/db-restore.ps1 -Mode sql -SqlDatabase "MyApp_Test" -InputFile "D:\backups\MyApp_Prod_20260612.bak" -IAmSure

# Загрузка .dt в файловую базу
powershell.exe -NoProfile -File .claude/skills/db-restore/scripts/db-restore.ps1 -Mode dt -InfoBasePath "C:\Bases\MyDB_Copy" -UserName "Admin" -PasswordEnv "MYDB_PASSWORD" -InputFile "D:\backups\MyDB.dt" -IAmSure
```
