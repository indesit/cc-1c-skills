# /log-analyze — Анализ журнала регистрации

Читает серверный журнал регистрации (старый текстовый формат: `1Cv8.lgf` + дневные `*.lgp` в `srvinfo\reg_<порт>\<uuid базы>\1Cv8Log`) и строит сводку: количество по важности, топ событий, топ пользователей, последние записи с комментариями. **Read-only.**

Новый формат `.lgd` (SQLite) пока не поддерживается — скрипт честно сообщит об этом.

## Команда

Движок — Python (`scripts/log-analyze.py`); ps1 — обёртка.

```powershell
python .claude/skills/log-analyze/scripts/log-analyze.py [параметры]
```

### Параметры

| Параметр | Обязательный | Описание |
|----------|:------------:|----------|
| `-LogDir <путь>` | нет | Каталог `1Cv8Log`; без него — автопоиск в srvinfo установленных платформ |
| `-Infobase <uuid>` | нет | Сузить автопоиск по UUID базы (см. `/srv-info`) |
| `-From / -To <yyyy-mm-dd>` | нет | Период (по умолчанию последние 7 дней) |
| `-Severity <список>` | нет | Фильтр: `E,W` (по умолчанию), `I`, `N` |
| `-Top <N>` | нет | Размер топов (10) |
| `-Details <N>` | нет | Сколько последних записей показать (20) |
| `-Json <файл>` | нет | Сохранить отчёт в JSON |

## После выполнения

Сводку покажи компактно. Если за период записей нет — провер: журнал регистрации может быть отключён в настройках базы (Администрирование → Настройка журнала регистрации) — сообщи об этом пользователю, это само по себе важная находка.

## Примеры

```powershell
# Ошибки и предупреждения за последнюю неделю
python .claude/skills/log-analyze/scripts/log-analyze.py

# Все события конкретного дня
python .claude/skills/log-analyze/scripts/log-analyze.py -From 2026-06-01 -To 2026-06-01 -Severity I,E,W,N
```
