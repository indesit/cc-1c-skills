# Программная работа с профилями групп доступа БСП

Применяется: BSL, БСП 3.x (ЗУП 3.1, БП 3.x, ERP 2.x, УТ 11.x). Программное создание/обновление справочника `ПрофилиГруппДоступа` через консоль кода или код обработчиков обновления.

---

## Тип реквизита `Роли.Роль` - НЕ строка

В БСП 3.x реквизит `Справочник.ПрофилиГруппДоступа.Роли.Роль` имеет тип **`СправочникСсылка.ИдентификаторыОбъектовМетаданных`**, а не строка с именем роли.

Если присвоить строку:
```bsl
// НЕПРАВИЛЬНО - значение молча отбрасывается
НоваяСтрока = Профиль.Роли.Добавить();
НоваяСтрока.Роль = "ПолныеПрава";
```

Симптом: `Профиль.Записать()` проходит без ошибок, но в ТЧ оказывается 0-1 строка вместо ожидаемого N. БСП-обработчик при записи фильтрует невалидные значения.

---

## Правильный путь - ссылка на ИдентификаторыОбъектовМетаданных

`ПолноеИмя` записи справочника для роли: `"Роль." + ИмяРоли`.

Массовое получение ссылок через запрос (предпочтительно для пакетной обработки):

```bsl
ПолныеИмена = Новый Массив;
Для Каждого ИмяРоли Из ИменаРолей Цикл
    ПолныеИмена.Добавить("Роль." + ИмяРоли);
КонецЦикла;

Запрос = Новый Запрос;
Запрос.Текст =
"ВЫБРАТЬ РАЗРЕШЕННЫЕ
|	Идентификаторы.ПолноеИмя КАК ПолноеИмя,
|	Идентификаторы.Ссылка КАК Ссылка
|ИЗ
|	Справочник.ИдентификаторыОбъектовМетаданных КАК Идентификаторы
|ГДЕ
|	Идентификаторы.ПолноеИмя В (&ПолныеИмена)
|	И НЕ Идентификаторы.ПометкаУдаления";
Запрос.УстановитьПараметр("ПолныеИмена", ПолныеИмена);

КартаИдентификаторов = Новый Соответствие;
Выборка = Запрос.Выполнить().Выбрать();
Пока Выборка.Следующий() Цикл
    КартаИдентификаторов.Вставить(Выборка.ПолноеИмя, Выборка.Ссылка);
КонецЦикла;
```

Точечное получение ссылки (БСП API, создаёт запись справочника при отсутствии):
```bsl
СсылкаИдент = ОбщегоНазначения.ИдентификаторОбъектаМетаданных(Метаданные.Роли["ИмяРоли"]);
```

---

## Если ссылок нет в справочнике

Если для какой-то роли ссылка в `ИдентификаторыОбъектовМетаданных` отсутствует (новая роль добавлена в конфигурацию, но БСП не успел зарегистрировать) - запустить обработчик обновления типовой подсистемы БСП "Идентификаторы объектов метаданных" (модуль `ОбновлениеВерсииИБ` / `ОбщегоНазначенияБТСПереопределяемый`) или вызвать `ОбщегоНазначения.ИдентификаторОбъектаМетаданных()` - он создаст запись.

## Шаблон создания/обновления профиля

```bsl
УстановитьПривилегированныйРежим(Истина);

ИмяПрофиля = "Имя профиля";

// 1. Собрать имена ролей в ИменаРолей (Массив)
// 2. Получить ссылки на идентификаторы (см. блок выше) -> КартаИдентификаторов

// 3. Найти/создать профиль
СсылкаПрофиля = Справочники.ПрофилиГруппДоступа.НайтиПоНаименованию(ИмяПрофиля, Истина);
Если СсылкаПрофиля.Пустая() Тогда
    Профиль = Справочники.ПрофилиГруппДоступа.СоздатьЭлемент();
    Профиль.Наименование = ИмяПрофиля;
Иначе
    Профиль = СсылкаПрофиля.ПолучитьОбъект();
    Если Профиль.ПоставляемыйПрофиль Тогда
        ВызватьИсключение "Поставляемый профиль - редактирование запрещено";
    КонецЕсли;
КонецЕсли;

// 4. Заполнить ТЧ Роли ссылками (НЕ строками)
Профиль.Роли.Очистить();
Для Каждого ИмяРоли Из ИменаРолей Цикл
    СсылкаИдент = КартаИдентификаторов.Получить("Роль." + ИмяРоли);
    Если СсылкаИдент = Неопределено Тогда
        Продолжить;
    КонецЕсли;
    НоваяСтрока = Профиль.Роли.Добавить();
    НоваяСтрока.Роль = СсылкаИдент;
КонецЦикла;

// 5. Назначение (Пользователи / ВнешниеПользователи) - заполнить, если пусто
Если Профиль.Назначение.Количество() = 0 Тогда
    Назн = Профиль.Назначение.Добавить();
    Назн.ТипПользователей = Справочники.Пользователи.ПустаяСсылка();
КонецЕсли;

// 6. Запись - БЕЗ ОбменДанными.Загрузка = Истина
// (иначе БСП-обработчики не отработают - регистрация в подсистеме УправлениеДоступом, обновление кэша прав)
Профиль.Записать();
```

## Сопоставление "имена ролей по синониму"

Если входные данные - синонимы ролей (русские названия из xlsx, документации), а не имена для разработчика:

```bsl
КартаПоСинониму = Новый Соответствие;
КартаПоСинонимуНорм = Новый Соответствие;
Для Каждого МетаРоль Из Метаданные.Роли Цикл
    КартаПоСинониму.Вставить(МетаРоль.Синоним, МетаРоль.Имя);

    Норм = НРег(СокрЛП(МетаРоль.Синоним));
    Пока СтрНайти(Норм, "  ") > 0 Цикл
        Норм = СтрЗаменить(Норм, "  ", " ");
    КонецЦикла;
    КартаПоСинонимуНорм.Вставить(Норм, МетаРоль.Имя);
КонецЦикла;
```

Двухуровневый матчинг: точное совпадение по `Синоним`, при промахе - по нормализованному (нижний регистр + СокрЛП + сжатие двойных пробелов). Покрывает разнобой в исходных данных.

---

## Анти-паттерны

- **Присвоение строки в `Роль`** - молча игнорируется БСП-обработчиком при записи.
- **`Профиль.ОбменДанными.Загрузка = Истина` перед `Записать()`** - отключает БСП-обработчики (регистрация в УправлениеДоступом, обновление прав), профиль записывается, но в системе прав не регистрируется.
- **Редактирование `ПоставляемыйПрофиль = Истина`** - запись возможна, но при следующем обновлении конфигурации БСП восстановит исходное состояние из поставки. Нужно создать пользовательский профиль с другим именем.
- **`УстановитьПривилегированныйРежим` без выключения** - стандартное правило, но в разовой консоли кода не критично.
