# Формирование отчёта 1С через прямое выполнение запроса (без СКД-движка)

Альтернатива стандартному выводу СКД (`ПроцессорВыводаРезультатаКомпоновкиДанныхВТабличныйДокумент` / `...ВКоллекциюЗначений`): получить исполняемый макет компоновки, извлечь из него текст SQL-запроса и параметры, выполнить как обычный `Запрос` и вывести результат вручную.

Применяется: когда нужно избежать высокого потребления памяти СКД-движком на больших отчётах (СКД держит в памяти все промежуточные данные обработки), либо нужен «голый» результат запроса плоской таблицей, либо альтернативная выгрузка (xlsx) рядом со штатным «Сформировать».

Не применять: когда нужны пользовательские группировки, условное оформление, расшифровки, итоги СКД - всё это даёт только штатный СКД-вывод.

---

## Алгоритм формирования

```bsl
// 1. Получить исполняемый макет компоновки.
//    Вариант А (платформенный, без побочных эффектов):
КомпоновщикМакета = Новый КомпоновщикМакетаКомпоновкиДанных;
МакетКомпоновки = КомпоновщикМакета.Выполнить(
    СхемаКомпоновки, НастройкиОтчета, , ,
    Тип("ГенераторМакетаКомпоновкиДанныхДляКоллекцииЗначений"));
//    Вариант Б (обёртка конфигурации, напр. ЗУП - умеет постобработку):
//    МакетКомпоновки = ЗарплатаКадрыОтчеты.МакетКомпоновкиДанных(СхемаКомпоновки, НастройкиОтчета, Неопределено);

// 2. (опционально) Постобработка макета конфигурацией - встраивание доп. данных в наборы.

// 3. Извлечь текст запроса основного набора данных (без тернарного оператора).
ОбъектЗапросаМакета = МакетКомпоновки.НаборыДанных.НаборДанных1.Запрос;
Если ТипЗнч(ОбъектЗапросаМакета) = Тип("Строка") Тогда
    ТекстЗапроса = ОбъектЗапросаМакета;
Иначе
    ТекстЗапроса = ОбъектЗапросаМакета.Текст;
КонецЕсли;

// 4. Параметры из скомпилированного макета (уже с учётом настроек).
Запрос = Новый Запрос(ТекстЗапроса);
Для Каждого ЗначениеПараметра Из МакетКомпоновки.ЗначенияПараметров Цикл
    Запрос.УстановитьПараметр(ЗначениеПараметра.Имя, ЗначениеПараметра.Значение);
КонецЦикла;

// 5. Внешние наборы (DataSetObject) - залить во временную таблицу под тем же именем.
Если СтрНайти(ТекстЗапроса, "ИмяВнешнегоНабора") > 0 Тогда
    МенеджерВТ = Новый МенеджерВременныхТаблиц;
    // СоздатьВТПоТаблицеЗначений или вручную ПОМЕСТИТЬ
    Запрос.МенеджерВременныхТаблиц = МенеджерВТ;
КонецЕсли;

// 6. Потоковый вывод (память O(1) на строку - НЕ Выгрузить()).
РезультатЗапроса = Запрос.Выполнить();
Выборка = РезультатЗапроса.Выбрать();
Пока Выборка.Следующий() Цикл
    // заполнение области табличного документа + накопление итогов
КонецЦикла;
```

### Источники запроса и параметров - два уровня

| Уровень | Что брать | Когда |
|---------|-----------|-------|
| Из **схемы** (`СхемаКомпоновки.НаборыДанных.НаборДанных1.Запрос` + `Схема.Параметры`) | Сырой текст запроса, параметры устанавливаются вручную из настроек (период/отбор) | Простые отчёты без сложных группировок |
| Из **скомпилированного макета** (`МакетКомпоновки.НаборыДанных.НаборДанных1.Запрос` + `Макет.ЗначенияПараметров`) | SQL с уже встроенными отборами/параметрами СКД | Сложные отчёты, нужен автосбор всех значений параметров |

Схему получать через `ОбъектОтчет.ПолучитьМакет("ИмяСхемы")` (независимая копия из метаданных), не через прямой реквизит объекта - безопаснее для побочных эффектов.

## Маршрут команды (кнопка в форме отчёта БСП)

Для внешних/внутренних отчётов на типовой форме `ВариантыОтчетов.ФормаОтчета` (без собственной формы). Подробности БСП-диспетчеризации - в memory проектов (паттерн добавления кнопки в форму отчёта).

1. **Модуль объекта отчёта** - `ОпределитьНастройкиФормы` → `Настройки.События.ПриСозданииНаСервере = Истина`; `ПриСозданииНаСервере` добавляет команду через `Форма.Команды.Добавить(...)` (`Действие = "Подключаемый_Команда"`) + `ОтчетыСервер.ВывестиКоманду(Форма, Команда, "Главное")`.
2. **`ОтчетыКлиентПереопределяемый.ОбработчикКоманды`** - ветка по `ФормаОтчета.ИмяФормы` + `Команда.Имя` → клиентский метод.
3. **Серверный код формирования - в МОДУЛЕ ФОРМЫ** (`CommonForm.ФормаОтчета`), не в общем серверном модуле: только в модуле формы доступен `РеквизитФормыВЗначение("Отчет")` для получения объекта отчёта на сервере.

```bsl
&НаСервере
Функция СформироватьЧерезПрямойЗапрос()
    ОбъектОтчет = РеквизитФормыВЗначение("Отчет");
    СхемаКомпоновкиДанных = ОбъектОтчет.ПолучитьМакет("ОсновнаяСхемаКомпоновкиДанных");
    // ВНИМАНИЕ: НЕ называть переменную НастройкиОтчета - это реквизит формы! (см. 1c-form-reserved-names.md)
    НастройкиДанные = ОбъектОтчет.КомпоновщикНастроек.ПолучитьНастройки();
    Возврат ОбъектОтчет.СформироватьПлоскуюТаблицу(СхемаКомпоновкиДанных, НастройкиДанные);
КонецФункции
```

Клиент-серверный обмен из общего модуля: передавать только **сериализуемые** данные (НастройкиКомпоновкиДанных сериализуются, форма как объект - нет), результат возвращать через временное хранилище (`ПоместитьВоВременноеХранилище` / `ПолучитьИзВременногоХранилища`).

## Подводные камни

- **`СвойстваРезультата` ломается**: если серверный метод формы перезаписал реквизит `НастройкиОтчета` (одноимённая локальная переменная) - клик по таб.документу падает. См. `1c-form-reserved-names.md`.
- **Тип `НаборДанных1.Запрос`** платформенно может быть строкой или объектом с `.Текст` - проверять `ТипЗнч` (без тернарного оператора - правило проекта).
- **Имя свойства параметра** в `МакетКомпоновки.ЗначенияПараметров` (`Имя` vs `ИмяПараметра`) уточнять при отладке.
- **Внешние наборы** (`DataSetObject`) при прямом выполнении не подключаются автоматически - заливать во временную таблицу под именем, под которым набор используется в SQL.
- **Постобработка данных конфигурации** (напр. `УчетРабочегоВремениРасширенный.ПолученияДанныхУчетаВремениПослеКомпоновкиМакета` в ЗУП) работает со **скомпилированным макетом** - доступна только при варианте Б (обёртка конфигурации), не при сыром запросе из схемы.
- **Память**: использовать `РезультатЗапроса.Выбрать()` + `Выборка.Следующий()`, НЕ `Выгрузить()` - иначе весь результат в памяти. Сам ТабличныйДокумент всё равно растёт - для очень больших объёмов рассмотреть вывод по частям.
- **Семантика СКД теряется**: плоский вывод не воспроизводит группировки/оформление/расшифровки. При сверке со штатным путём сравнивать только данные и ресурсные итоги, не внешний вид.
