# 1C Extension Patterns (CFE)

BSL-паттерны и правила работы с расширениями конфигурации 1С.

Применяется: при работе с кодом расширений (`**/Extensions/**/*.bsl`) и с XML-метаданными ролей расширения.

---

## Типы перехватчиков

| Декоратор | Тип | Когда использовать |
|-----------|-----|-------------------|
| `&Перед("ИмяМетода")` | Before | Код до вызова оригинального метода |
| `&После("ИмяМетода")` | After | Код после вызова оригинального метода |
| `&ИзменениеИКонтроль("ИмяМетода")` | ModificationAndControl | Полная замена тела метода |

### Before / After — простые перехватчики

```bsl
&НаСервере
&Перед("ПриЗаписи")
Процедура Расш1_ПриЗаписи()
    // Код выполняется ДО оригинального ПриЗаписи
КонецПроцедуры

&НаСервере
&После("ПриЗаписи")
Процедура Расш1_ПослеЗаписи()
    // Код выполняется ПОСЛЕ оригинального ПриЗаписи
КонецПроцедуры
```

### ИзменениеИКонтроль — полная замена

```bsl
&НаСервере
&ИзменениеИКонтроль("ОбработкаЗаполнения")
Процедура Расш1_ОбработкаЗаполнения(ДанныеЗаполнения, СтандартнаяОбработка)
    #Удаление
    // Оригинальный код (удалённый)
    #КонецУдаления

    #Вставка
    // Новый код (вставленный)
    #КонецВставки

    ПродолжитьВызов();
КонецПроцедуры
```

---

## Правила ПродолжитьВызов()

- `&Перед` — `ПродолжитьВызов()` вызывается автоматически после, НЕ вызывать вручную
- `&После` — оригинал уже выполнен, `ПродолжитьВызов()` не используется
- `&ИзменениеИКонтроль` — **обязательно** вызвать `ПродолжитьВызов()` для выполнения оригинала. Без вызова — оригинальный метод не выполнится

---

## Маркеры изменений

Маркеры обязательны внутри `&ИзменениеИКонтроль` для отслеживания изменений:

| Маркер | Назначение |
|--------|-----------|
| `#Вставка` / `#КонецВставки` | Новый код, добавленный расширением |
| `#Удаление` / `#КонецУдаления` | Оригинальный код, который был заменён |

---

## Ограничения заимствованных объектов

- Заимствованный объект (`ObjectBelonging=Adopted`) — копия метаданных из основной конфигурации
- Нельзя удалять существующие реквизиты/ТЧ заимствованного объекта
- Можно добавлять собственные реквизиты/ТЧ (с префиксом расширения)
- Модули заимствованных объектов — только перехватчики, не прямое редактирование
- Формы заимствованных объектов — можно добавлять элементы, нельзя удалять существующие

## Антипаттерны расширений

### Прямое изменение заимствованного модуля

```bsl
// НЕПРАВИЛЬНО: редактирование оригинального кода напрямую
Процедура ПриЗаписи()
    // изменённый код...
КонецПроцедуры

// ПРАВИЛЬНО: перехватчик
&Перед("ПриЗаписи")
Процедура Расш1_ПриЗаписи()
    // дополнительный код
КонецПроцедуры
```

### Забытый ПродолжитьВызов

```bsl
// ОПАСНО: оригинальный метод не выполнится!
&ИзменениеИКонтроль("ОбработкаПроведения")
Процедура Расш1_ОбработкаПроведения(Отказ)
    // свой код...
    // ЗАБЫЛИ: ПродолжитьВызов();
КонецПроцедуры
```

### Именование без префикса

```bsl
// Плохо: конфликт имён с другими расширениями
Процедура ДополнительнаяПроверка()

// Хорошо: префикс расширения
Процедура МоёРасш_ДополнительнаяПроверка()
```

---

## Назначение расширений

| Тип | Purpose | Когда использовать |
|-----|---------|-------------------|
| Исправление | `Patch` | Минимальные изменения, только перехватчики |
| Доработка | `Customization` | Реквизиты, формы, модули |
| Дополнение | `AddOn` | Полноценный новый функционал |

---

## Права в Rights.xml роли расширения

Правила прав в ролях универсальны (не только для расширений) — см. `1c-role-rights.md`.

Ключевое для расширений:
- Не включать `Enum.*` в Rights.xml роли — перечисления общедоступны, явные права избыточны и ломают `LoadConfigFromFiles -Extension` (платформа 8.3.27)
- Если добавляешь новый объект метаданных + права на него в роли в одной выгрузке — рассмотри двухпроходную загрузку (сначала без роли, потом с ролью), чтобы избежать зависания резолва прав
