# Структура исходников v8unpack

Формат исходников после распаковки бинарных файлов 1С утилитой `saby v8unpack`. Применяется при чтении, модификации и создании объектов метаданных в распакованных конфигурациях (CF/CFE/EPF).

---

## Корень конфигурации

```
src/
  Configuration.json          ← метаданные конфигурации
  Configuration.seance.bsl    ← модуль сеанса
  Configuration.app.bsl       ← модуль управляемого приложения
  Configuration.802.bsl       ← модуль обычного приложения
  Configuration.con.bsl       ← модуль внешнего соединения
  Configuration.4.json        ← доп. данные конфигурации
  help.html                   ← справка
  Заставка                    ← заставка (двоичный файл)
  Catalog/                    ← справочники
  Document/                   ← документы
  CommonModule/               ← общие модули
  InformationRegister/        ← регистры сведений
  AccumulationRegister/       ← регистры накопления
  Report/                     ← отчёты
  DataProcessor/              ← обработки
  Enum/                       ← перечисления
  ...                         ← другие типы метаданных
```

### Configuration.json

```json
{
  "v8unpack": "1.2.6",
  "file_uuid": "1e4190e9-...",
  "root": [["2", "1e4190e9-..."]],
  "uuid": "6b021b70-...",
  "name": "УправлениеТорговлей",
  "name2": {"ru": "Управление торговлей, редакция 10.3"},
  "comment": "",
  "header": [...]
}
```

- `v8unpack` — версия утилиты (проверяется при сборке)
- `name` — программное имя конфигурации
- `name2` — синонимы по языкам
- `header` — сырые данные (вложенные массивы, не редактировать вручную)

## Типы файлов

### *.id.json — UUID объекта

```json
{"uuid": "5a53ae4f-f7b9-4bed-93d1-1309a746b20d"}
```

Вынесен отдельно от основного .json, чтобы одни исходники могли порождать разные продукты с разными UUID.

### *.json — метаданные объекта

```json
{
  "name": "ПоступлениеТоваров",
  "name2": {"ru": "Поступление товаров и услуг"},
  "comment": "(Общ)",
  "obj_version": "38",
  "header": [...]
}
```

- `name` — программное имя
- `name2` — синонимы по языкам
- `comment` — комментарий
- `obj_version` — версия формата объекта метаданных
- `header` — сырые данные метаданных (не редактировать вручную)

### *.obj.bsl — модуль объекта / формы

Обычный BSL-код в кодировке UTF-8 BOM. Может содержать include-области:

```bsl
#Область include_core_common_utils
    // код из ../core/common/utils.bsl
#КонецОбласти
```

### *.mgr.bsl — модуль менеджера

BSL-код модуля менеджера. Аналогично `*.obj.bsl`.

### Коды модулей по типам объектов

| Тип объекта | Суффиксы модулей |
|-------------|------------------|
| Configuration | `.seance.bsl` (сеанса), `.app.bsl` (управл. приложения), `.802.bsl` (обычн. приложения), `.con.bsl` (внешн. соединения) |
| Catalog, Document, DataProcessor | `.obj.bsl` (объекта), `.mgr.bsl` (менеджера) |
| CommonModule | `.obj.bsl` (код модуля) |
| *Form | `.obj.bsl` (модуль формы) |

## Формы — подробная структура

### Файлы формы

Каждая форма — именованная папка с 4 файлами:

| Файл | Содержимое |
|------|-----------|
| `*.id.json` | UUID формы |
| `*.json` | Метаданные формы + тип + версия |
| `*.obj.bsl` | Модуль формы (BSL-код) |
| `*.elem.json` | Элементы формы (дерево + данные + реквизиты + команды + параметры) |

### Тип формы (*.json)

```json
{
  "name": "Форма",
  "name2": {"ru": "Форма"},
  "Тип формы": "0",
  "Версия элементов формы": "0-27",
  "form": [...],
  "header": [...]
}
```

| Значение `Тип формы` | Тип |
|-----------------------|-----|
| `"0"` | Обычная форма (ОФ) |
| `"1"` | Управляемая форма (УФ) |

| Значение `Версия элементов формы` | Платформа |
|------------------------------------|-----------|
| `"0-26"` | Обычная форма 8.0/8.1 |
| `"0-27"` | Обычная форма 8.2 |
| `"1"` | Управляемая форма 8.3+ |

### Формат *.elem.json

```json
{
  "params": [],           // параметры формы (только УФ)
  "props": [...],         // реквизиты формы
  "commands": [],         // команды формы (только УФ)
  "tree": [...],          // дерево элементов (иерархия)
  "data": {...}           // данные элементов (плоский словарь)
}
```

### Секция props — реквизиты формы

```json
"props": [
  {
    "name": "ОбработкаОбъект",
    "id": "0",
    "raw": [["0"], "0", "0", "1", "\"ОбработкаОбъект\"",
            ["\"Pattern\"", ["\"#\"", "uuid-типа"]]]
  },
  {
    "name": "НачПериода",
    "id": "5",
    "raw": [["5"], "1", "0", "1", "\"НачПериода\"",
            ["\"Pattern\"", ["\"D\""]]]
  }
]
```

- `name` — имя реквизита
- `id` — числовой ID (используется для привязки элементов)
- `raw` — сырые данные (формат платформы)

Типы данных в `raw[5][1]`:
- `["\"#\"", "uuid"]` — ссылочный тип (Справочник, Документ)
- `["\"D\""]` — Дата
- `["\"S\""]` — Строка
- `["\"N\""]` — Число
- `["\"B\""]` — Булево

### Секция tree — дерево элементов

**Управляемые формы (УФ):**
```json
"tree": [
  {"name": "Номер", "type": "Field"},
  {"name": "СписокТоваров", "type": "Table",
   "child": [
     {"name": "СписокТоваров/Номенклатура", "type": "Field"},
     {"name": "СписокТоваров/Количество", "type": "Field"}
   ]},
  {"name": "ГруппаНастроек", "type": "Group",
   "child": [
     {"name": "ФлагОтбора", "type": "Field"}
   ]}
]
```

Типы элементов УФ: `Field`, `Group`, `Table`, `Button`, `Decoration`, `ItemAddition`.

**Обычные формы (ОФ):**
```json
"tree": [
  {"name": "Производитель", "type": "SelectField"},
  {"name": "НачПериода", "type": "InputField"},
  {"name": "ПолеТекста", "type": "TextBox"}
]
```

Типы элементов ОФ: `Panel`, `InputField`, `SelectField`, `TextBox`, `CheckBox`, `RadioBtn`, `CommandPanel`, `Label`, `Separator`, `CommandBar`, `Chart`, `SpreadSheet` и др.

### Секция data — данные элементов

Плоский словарь. Ключ = путь через `/` (соответствует позиции в tree):

```json
"data": {
  "Номер": {"raw": [...], "ver": 4},
  "СписокТоваров": {"raw": [...], "ver": 4},
  "СписокТоваров/Номенклатура": {"raw": [...], "ver": 4, "ПутьКДанным": "Объект.Товары.Номенклатура"},
  "ГруппаНастроек/ФлагОтбора": {"raw": [...], "ver": 4, "ПутьКДанным": "ФлагОтбора"}
}
```

- `raw` — сырые данные элемента (формат платформы)
- `ver` — версия декодера (4 = УФ, 27 = ОФ 8.2, 26 = ОФ 8.0)
- `ПутьКДанным` — привязка к реквизиту (если есть), формат: `Объект.ТЧ.Реквизит` или `ИмяРеквизита`
- `ИмяКоманды` — привязка к команде (для кнопок)

## Механизм include-областей

### В BSL-файлах

```bsl
#Область include_core_form3_test
    // код сохраняется в ../core/form3/test.bsl
#КонецОбласти
```

При распаковке код вырезается в отдельный файл. При сборке вставляется обратно.

- `include_*` — редактируемая область (обновляется при распаковке)
- `includr_*` — read-only область (не обновляется, только подставляется при сборке)

### В элементах форм

Элементы с именем `include_*` или `includr_*` и их дочерние выносятся в отдельный `.elem.json` файл.

## Работа с формами — практика

### Чтение формы

1. Файл `*.json` → тип формы (`Тип формы`) и версия (`Версия элементов формы`)
2. Файл `*.elem.json` → реквизиты (`props`), элементы (`tree` + `data`)
3. Файл `*.obj.bsl` → код модуля формы

### Модификация кода формы

Редактировать `*.obj.bsl` — обычный BSL-файл. После изменения — собрать обратно через `v8unpack -B`.

### Модификация элементов формы

**ВАЖНО:** Поля `header`, `raw`, `form` содержат сырые данные в формате платформы 1С. Ручное редактирование этих полей крайне рискованно и не рекомендуется — результат может не загрузиться в платформу.

Безопасно редактировать:
- `*.obj.bsl` — код модуля формы
- `name`, `name2`, `comment` в `*.json` — метаданные формы

Не редактировать вручную:
- `header`, `form` в `*.json`
- `raw` в `*.elem.json`
- `tree` / `data` в `*.elem.json` (только для анализа, не для ручной правки)

### Создание новых объектов

v8unpack НЕ предназначен для создания новых объектов метаданных с нуля. Он работает с существующими бинарниками. Для создания новых объектов:
1. Создать объект в конфигураторе / EDT
2. Выгрузить в CF
3. Распаковать v8unpack
4. Далее модифицировать код (`.bsl`)

Или при наличии шаблонного объекта того же типа — скопировать папку, заменить UUID в `*.id.json`, изменить `name`/`name2` в `*.json`, и собрать. Но `header` и `raw` должны быть от корректного объекта того же типа.

## Режим descent — версионирование расширений

Суффикс версии конфигурации в формате `XYYZZZ` (4 разряда × 3 знака, первый — произвольная длина):

| Версия конфигурации | Суффикс |
|---------------------|---------|
| 4.10.2 | 4010002 |
| 3.0.112 | 3000112 |
| 2.5.6 | 2005006 |

Файлы с суффиксом: `DocumentForm.obj.4010002.bsl` — код для версии 4.10.2.

При сборке с `--descent X` берётся файл с наибольшим суффиксом ≤ X.

## index.json — маппинг файлов

```json
{
  "ExternalDataProcessor.json": "core/ExternalDataProcessor.json",
  "Form": {
    "API": {
      "API.obj.bsl": "core/Form/API.obj.bsl",
      "*": "core/Form/API"
    }
  },
  "index.json": ["submodule/index.json"]
}
```

- Конкретный файл → конкретный путь
- `"*"` → все файлы в папке
- `null` → исключение из `*`
- Массив → подключить вложенные index.json
