# 1C Role Rights — правила прав в ролях

Применяется: при создании/правке ролей (`Roles/<RoleName>/Ext/Rights.xml`), при работе с разграничением доступа в любой конфигурации 1С (типовые конфигурации, самописные, расширения).

---

## Перечисления (Enum) — НЕ давать прав в ролях

**Не включать `Enum.<ИмяПеречисления>` в Rights.xml ролей.**

В 1С перечисления — это объекты с фиксированным набором значений, доступные на чтение всем пользователям без ограничения. Платформа считает их "общедоступными": явные права `Read` / `View` на них не нужны и могут ломать загрузку расширений (см. ниже).

```xml
<!-- НЕ НАДО ни в роли расширения, ни в роли основной конфигурации -->
<object>
    <name>Enum.СтатусыОбработки</name>
    <right><name>Read</name><value>true</value></right>
    <right><name>View</name><value>true</value></right>
</object>
```

### Почему

- Перечисления имеют доступ без ограничений — явные права избыточны.
- Загрузка расширений через `1cv8.exe DESIGNER /LoadConfigFromFiles -Extension` (платформа 8.3.27) **виснет** при наличии блока прав на новый Enum в Rights.xml роли. /Out пустой 60+ сек, без ошибки.
- EDT 2026.1 при ручном включении Enum в права роли создаёт этот блок при выгрузке — нужно убирать перед сохранением роли в EDT, или вручную править Rights.xml перед загрузкой.

### Как применять

Когда читаешь/правишь `Rights.xml` роли — проверить наличие `<name>Enum.*</name>` блоков и предложить пользователю удалить их (с пояснением: перечисления доступны всем, права не нужны, и это может ломать загрузку расширения).

---

## Ограничения прав на объекты, добавляемые в той же транзакции (расширения)

Для расширений — если добавляешь новый объект метаданных (Catalog, Document, InformationRegister) + роль с правами на него в одной выгрузке, порядок резолва прав иногда вызывает зависание `LoadConfigFromFiles`.

**Workaround:** двухпроходная загрузка — сначала расширение без роли (или с минимальной ролью), затем полная выгрузка с правами.

---

## Pre-flight check перед загрузкой расширения

Перед `1cv8.exe DESIGNER /LoadConfigFromFiles -Extension` проверить `Roles/<RoleName>/Ext/Rights.xml`:

```powershell
# Pre-flight: поиск проблемных Enum-прав в ролях
Get-ChildItem -Path "$ConfigDir\Roles" -Recurse -Filter "Rights.xml" | ForEach-Object {
    $content = Get-Content $_.FullName -Raw
    if ($content -match '<name>Enum\.[^<]+</name>') {
        Write-Warning "Enum-права в $($_.FullName) — могут вызвать зависание LoadConfigFromFiles"
    }
}
```
