# Асинхронные методы 1С (Асинх / Ждать / Обещание)

Правила использования нового механизма асинхронности (платформа 8.3.18+).

Применяется: при написании клиентского кода с асинхронными вызовами.

Источник: [Шпаргалка по функциям АСИНХ](https://infostart.ru/1c/articles/1702526/)

---

## Основные принципы

1. Все новые асинхронные функции (суффикс `Асинх`) возвращают объект `Обещание`
2. `Ждать` — оператор ожидания завершения `Обещание`. Код после `Ждать` не выполняется до завершения обещания
3. Вызов методов `*Асинх` возможен **только внутри** процедур/функций с ключевым словом `Асинх`
4. `Обещание` имеет 3 состояния: **Ожидание**, **Успех**, **Провал**
5. Работает только **на клиенте** (`&НаКлиенте`)

---

## Соответствие старых и новых методов

Предпочитай новый синтаксис `*Асинх` вместо `Показать*` с `ОписаниеОповещения`:

| Модальный (запрещён) | Через оповещение | Новый Асинх |
|---|---|---|
| `Предупреждение()` | `ПоказатьПредупреждение()` | `ПредупреждениеАсинх()` |
| `Вопрос()` | `ПоказатьВопрос()` | `ВопросАсинх()` |
| `ОткрытьЗначение()` | `ПоказатьЗначение()` | `ОткрытьЗначениеАсинх()` |
| `ВвестиЧисло()` | `ПоказатьВводЧисла()` | `ВвестиЧислоАсинх()` |
| `НайтиФайлы()` | `НачатьПоискФайлов()` | `НайтиФайлыАсинх()` |
| `ПоместитьФайлы()` | `НачатьПомещениеФайлов()` | `ПоместитьФайлыНаСерверАсинх()` |

---

## Возвращаемые значения (результат Ждать)

| Метод | Результат `Ждать Обещание` |
|---|---|
| `ПредупреждениеАсинх` | `Неопределено` |
| `ВопросАсинх` | `КодВозвратаДиалога` |
| `ОткрытьЗначениеАсинх` | `Неопределено` |
| `ВвестиЧислоАсинх` | `Число` |
| `НайтиФайлыАсинх` | `Массив` объектов типа `Файл` |
| `ПоместитьФайлыНаСерверАсинх` | `Массив` из `ОписаниеПомещенногоФайла` (успех) / `Неопределено` (провал) |

## Базовый шаблон

```bsl
&НаКлиенте
Асинх Процедура МояПроцедура()

    Обещание = ВопросАсинх("Продолжить?", РежимДиалогаВопрос.ДаНет);
    Результат = Ждать Обещание;

    Если Результат = КодВозвратаДиалога.Да Тогда
        // Действие
    КонецЕсли;

КонецПроцедуры
```

## Критические правила

### 1. Без `Ждать` исключения теряются МОЛЧА

Если вызвать асинхронную функцию без `Ждать`, и внутри возникнет исключение — оно будет **проглочено без следа**. Код продолжит выполнение как ни в чём не бывало.

```bsl
// ОПАСНО: исключение внутри ВызватьОшибку() будет потеряно!
Процедура МояПроцедура(Команда)
    ВызватьОшибку(); // Асинх-функция вызвана без Ждать
КонецПроцедуры

Асинх Функция ВызватьОшибку()
    ВызватьИсключение "ошибка!"; // никто не увидит
КонецФункции
```

**Правило:** Всегда используй `Ждать` при вызове `Асинх`-функций, если важен результат или обработка ошибок. Без `Ждать` — только для fire-and-forget сценариев, когда осознанно игнорируешь результат и ошибки.

### 2. `Асинх` в обработчиках событий формы НЕ блокирует

Обработчики событий формы (`ПриОткрытии`, `ПередЗакрытием`) с ключевым словом `Асинх` **не работают как ожидается** — платформа не ждёт их завершения.

```bsl
// НЕ РАБОТАЕТ! Форма всё равно откроется, Отказ = Истина проигнорирован.
&НаКлиенте
Асинх Процедура ПриОткрытии(Отказ)
    Отказ = Истина;                    // платформа уже ушла дальше
    ВызватьИсключение "ошибка!";       // пользователь увидит ошибку, но форма уже открыта
КонецПроцедуры
```

**Правило:** НЕ делай обработчики событий формы (`ПриОткрытии`, `ПередЗакрытием`, `ПриЗаписиНаСервере` и т.д.) асинхронными. Параметр `Отказ` и любое влияние на поток выполнения формы будут проигнорированы. Вместо этого — вызывай отдельную `Асинх`-процедуру из обычного обработчика.

### 3. Обработчики команд — МОЖНО делать асинхронными

В отличие от обработчиков событий формы, обработчики команд **можно** безопасно объявлять как `Асинх`:

```bsl
&НаКлиенте
Асинх Процедура МояКоманда(Команда)
    Обещание = ВвестиЧислоАсинх(1, "Укажите число");
    Результат = Ждать Обещание;
КонецПроцедуры
```

## Паттерн: Вопрос при открытии формы

Обработчик `ПриОткрытии` — обычная процедура, вызывает отдельную `Асинх`-процедуру:

```bsl
&НаКлиенте
Процедура ПриОткрытии(Отказ)
    ЗадатьВопросПриОткрытии();
КонецПроцедуры

&НаКлиенте
Асинх Процедура ЗадатьВопросПриОткрытии()

    Обещание = ВопросАсинх("Продолжить работу?",
        РежимДиалогаВопрос.ДаНет, , КодВозвратаДиалога.Да);
    Результат = Ждать Обещание;

    Если Результат <> КодВозвратаДиалога.Да Тогда
        ЭтаФорма.Закрыть();
    КонецЕсли;

КонецПроцедуры
```

## Паттерн: Вопрос при закрытии формы

Используется переменная-флаг для предотвращения рекурсии:

```bsl
&НаКлиенте
Перем ВопросПриЗакрытииЗадан;

&НаКлиенте
Процедура ПередЗакрытием(Отказ, ЗавершениеРаботы,
        ТекстПредупреждения, СтандартнаяОбработка)

    Если НЕ ВопросПриЗакрытииЗадан Тогда
        ЗадатьВопросПриЗакрытии();
        Отказ = Истина;
    КонецЕсли;

КонецПроцедуры

&НаКлиенте
Асинх Процедура ЗадатьВопросПриЗакрытии()

    Обещание = ВопросАсинх("Закрыть форму?",
        РежимДиалогаВопрос.ДаНет, , КодВозвратаДиалога.Да);
    Результат = Ждать Обещание;

    Если Результат = КодВозвратаДиалога.Да Тогда
        ВопросПриЗакрытииЗадан = Истина;
        ЭтаФорма.Закрыть();
    КонецЕсли;

КонецПроцедуры

// Инициализация
ВопросПриЗакрытииЗадан = Ложь;
```

## Паттерн: Работа с файлами

```bsl
&НаКлиенте
Асинх Процедура ЗагрузитьФайлыНаСервер()

    // 1. Найти файлы
    Обещание = НайтиФайлыАсинх(ПутьКПапке, "*.*", Ложь);
    НайденныеФайлы = Ждать Обещание;

    // 2. Подготовить описания
    ДобавляемыеФайлы = Новый Массив;
    Для Каждого НайденныйФайл Из НайденныеФайлы Цикл
        Описание = Новый ОписаниеПередаваемогоФайла;
        Описание.Имя = НайденныйФайл.ПолноеИмя;
        ДобавляемыеФайлы.Добавить(Описание);
    КонецЦикла;

    // 3. Поместить на сервер
    Обещание = ПоместитьФайлыНаСерверАсинх(
        , , ДобавляемыеФайлы, ЭтаФорма.УникальныйИдентификатор);
    Результат = Ждать Обещание;

КонецПроцедуры
```
