# Правила рефакторинга 1С

При рефакторинге используется гибридный подход:

1. **Top-down анализ** — сначала понять бизнес-логику: явно построить всю цепочку вызовов и проследить поток данных.
2. **Bottom-up рефакторинг** — начать с низкоуровневых утилитарных функций.
3. **Интеграция** — встроить обновлённые компоненты в высокоуровневые процедуры.

---

## Когда рефакторить

| Критерий | Порог |
|----------|-------|
| Цикломатическая сложность | >15 (количество независимых путей выполнения) |
| Длина функции | >200 строк (см. также `code-review-checklist.md` Medium) |
| Дублирование кода | 3+ повторения одного блока |
| Уровень вложенности | >3 уровней Если/Цикл |
| Количество параметров | >5 параметров |
| God-модуль | >2000 строк в одном модуле |

Не рефакторить: код, который не затронут текущей задачей (если не запрошено явно).

---

## Техники рефакторинга

### 1. Извлечение метода

Выделить логически самостоятельный блок в отдельную процедуру/функцию.

```bsl
// До:
Процедура ОбработатьДокумент(Документ)
    // ... 50 строк валидации ...
    // ... 80 строк расчёта сумм ...
    // ... 30 строк записи движений ...
КонецПроцедуры

// После:
Процедура ОбработатьДокумент(Документ)
    ПроверитьКорректность(Документ);
    РассчитатьСуммы(Документ);
    ЗаписатьДвижения(Документ);
КонецПроцедуры
```

### 2. Упрощение условной логики (guard clauses)

Заменить вложенные `Если` на ранние выходы - см. `anti_patterns.md` п.11.

### 3. Консолидация дублирующего кода

Повторяющийся код (3+ раз) — вынести в общий метод с параметрами.

```bsl
// До: три одинаковых блока обработки результатов запроса
РезультатЗапроса = Запрос1.Выполнить();
Выборка = РезультатЗапроса.Выбрать();
Пока Выборка.Следующий() Цикл
    ОбработатьЗапись(Выборка);
КонецЦикла;

// (тот же блок для Запрос2, Запрос3...)

// После:
Процедура ОбработатьРезультатЗапроса(Запрос)
    РезультатЗапроса = Запрос.Выполнить();
    Выборка = РезультатЗапроса.Выбрать();
    Пока Выборка.Следующий() Цикл
        ОбработатьЗапись(Выборка);
    КонецЦикла;
КонецПроцедуры
```

### 4. Замена магических значений

Вынести литералы в именованные константы или параметры.

```bsl
// До:
Если ДнейПросрочки > 30 Тогда
    Штраф = Сумма * 0.01;
КонецЕсли;

// После:
МаксДнейБезШтрафа = 30;
СтавкаШтрафа      = 0.01;

Если ДнейПросрочки > МаксДнейБезШтрафа Тогда
    Штраф = Сумма * СтавкаШтрафа;
КонецЕсли;
```

### 5. Кеширование повторных вычислений в цикле

Использовать Соответствие как кеш - см. `anti_patterns.md` п.9.

### 6. Батчинг клиент-серверных переходов

Собрать данные на клиенте, передать одним вызовом на сервер - см. `anti_patterns.md` п.6.

### 7. Переиспользование существующего кода

Перед написанием нового кода - проверить общие модули и БСП (например, `ОбщегоНазначения.ЗначениеРеквизитаОбъекта` вместо запроса).

---

## Принципы

- **Сохранять поведение** — рефакторинг не меняет что делает код, только как
- **Атомарные шаги** — каждое изменение проверяемо отдельно
- **Ясность важнее краткости** — не превращать понятный код в "умный"
- **Не over-engineer** — не создавать абстракции для одноразовых операций
